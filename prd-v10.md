# Shorts Fission - 短视频裂变系统 PRD v1.0

## 1. 项目背景

### 1.1 业务需求

为体育赛事直播平台提供 TikTok 和 YouTube 矩阵账号推广方案。核心需求是：
- 从 YouTube/TikTok 下载体育赛事短视频（≤3分钟）
- 将 1 条元视频裂变为 10-20 条独特变体视频
- 支持账号矩阵发布，避免内容重复

### 1.2 痛点分析

| 痛点 | 说明 |
|------|------|
| **内容重复** | 直接搬运会导致平台查重、账号降权 |
| **效率低下** | 手动剪辑变体耗时巨大 |
| **规模瓶颈** | 无法支撑大规模账号矩阵 |
| **版权风险** | 需要二次创作规避风险 |

### 1.3 目标

- **输入**：1 条源短视频（≤3分钟）
- **输出**：10-20 条独特变体视频
- **效率**：全自动化，无需人工干预
- **质量**：每个变体都通过平台原创度检测

---

## 2. 解决方案

### 2.1 技术方案

基于以下开源项目组合：
- **ytb2bili**：视频下载、字幕提取、AI翻译
- **ai-mixed-cut**：视觉变体技术（LUT调色、动态特效）
- **video-mover**：实时监控、帧交换、频域扰乱

## 6. 技术栈

### 后端
- **框架**：FastAPI
- **任务队列**：Celery + Redis
- **数据库**：SQLite / PostgreSQL
- **视频处理**：FFmpeg + yt-dlp
- **语音识别**：WhisperX 3.3
- **AI**：Gemini 2.0 / DeepSeek

### 前端
- **框架**：React 18 + TypeScript
- **UI**：TailwindCSS / Ant Design
- **状态管理**：Zustand
- **实时通信**：WebSocket

### 部署
- **容器化**：Docker + Docker Compose
- **反向代理**：Nginx

## 8. 配置示例

```yaml
# config/config.yaml

app:
  name: "Shorts Fission"
  variant_count: 15  # 默认生成 15 个变体 (10-20)

# 目标语言配置
language:
  target: "en"
  translate_if_not_target: true

# 功能开关
features:
  subtitle_translation: false
  tts_redub: false
  ai_rewrite: false

# 下载配置
download:
  single:
    no_watermark: true
  batch:
    enabled: true
    max_concurrent: 3
  monitor:
    enabled: true
    check_interval: 300

# 视觉变体配置
visual:
  effects_pool:
    - lut_color
    - dynamic_zoom
    - color_shift_sbc
    - texture_noise
    - mask_overlay
    - frame_swap
    - blur_edge
    - blur_background
    - color_shift_rgb
    # ... 共 17 种
  min_effects: 1
  max_effects: 5

# 音频变体配置
audio:
  redub: false
  bgm_replace: true
  bgm_volume: 0.3

# 球类 BGM 映射
sports_bgm:
  baseball: "sports_bgm/baseball/"
  basketball: "sports_bgm/basketball/"
  football: "sports_bgm/football/"
  hockey: "sports_bgm/hockey/"
```
