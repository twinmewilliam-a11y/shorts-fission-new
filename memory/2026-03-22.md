## 📚 经验总结：批量任务并发优化

### 核心原则

**对于大批量执行的任务，需要梳理流程发现串行/并行任务，进行流程优化提升效率。**

### 分析方法

```
1. 列出所有步骤
2. 识别数据依赖关系
3. 区分：
   - 串行任务（有数据依赖）
   - 并行任务（无数据依赖）
4. 设计并发流程
```

### 实例：视频变体生成

| 步骤 | 类型 | 原因 | 耗时 |
|------|------|------|------|
| WhisperX 转录 | **串行** | 后续步骤依赖 words.json | 15秒 |
| Remotion 渲染 | **串行** | 后续步骤依赖字幕视频 | 60秒 |
| 视觉处理 + overlay | **并行** | 每个变体独立，无依赖 | 50秒 |

### 优化效果

| 模式 | 5个变体耗时 |
|------|-----------|
| 全串行 | ~5分钟 |
| 部分并发 | ~2分钟 |
| **提升** | **60%** |

### 通用公式

```
总耗时 = Σ串行步骤耗时 + MAX(并行步骤耗时)
      ≠ Σ所有步骤耗时
```

### 实现方式

```python
# 1. 串行步骤：主任务执行
words = extract_word_timestamps()  # 一次
subtitle_video = render_remotion(words)  # 一次

# 2. 并行步骤：拆分为子任务
for i in range(count):
    generate_single_variant.delay(
        video_id, i, subtitle_video
    )  # Celery Worker 并发执行
```

### 适用场景

- ✅ 视频批量处理
- ✅ 图片批量转换
- ✅ 数据批量分析
- ✅ 文件批量上传
- ✅ API 批量调用
